Implementation Guidelines
⌨️
Keyboard navigation — all interactive elements reachable
Every button, input, link, and custom control must be reachable and operable with Tab/Shift+Tab. Custom components must handle Enter/Space for activation and Escape for dismissal.
tabIndex="0" + role="button" + onKeyDown handler
🎯
Focus ring — never suppress, only style
Never use outline:none or outline:0 without providing a custom focus ring. MaV uses a 2px solid ring at --mav-primary with a 2px offset. Apply to :focus-visible not :focus to avoid showing on mouse clicks.
:focus-visible { outline: 2px solid var(--mav-primary); outline-offset: 2px; }
📢
Live regions for dynamic content
Error messages, loading states, notifications, and toast messages must use aria-live="polite" (non-urgent) or role="alert" (urgent). Never convey state through color alone.
<div role="alert" aria-live="assertive">Invalid OTP</div>
🏷️
All inputs must have an accessible label
Use <label for=> for visible labels, aria-label for icon-only controls, aria-labelledby for complex labelling. Placeholder text is not a substitute for a label — it disappears on input.
<label for="email">Email</label><input id="email">
📱
Touch targets — minimum 44×44px
All tap targets on mobile must be at least 44×44px (iOS) or 48×48dp (Android). If the visual element is smaller, expand the hit target with padding or a pseudo-element.
min-height: 44px; padding: 10px 16px;
